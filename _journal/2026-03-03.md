# 2026-03-03 — Project inception: spamprobe replacement in Rust

## Context

channonstreet.com email delivery failures on mrn led to investigating spamprobe
BDB corruption. The root cause: Dovecot sieve processes crash mid-write, leaving
BDB `__db.*` environment files corrupt. Subsequent spamprobe invocations deadlock
on the corrupt BDB, causing all mail delivery to that user to fail with LMTP
timeouts.

Fixed immediately by hardening `cleanspam` script on mrn:
- Auto-kill stuck spamprobe processes >5 min old
- Remove stale `__db.*` + `db5.3_recover` before reading database
- Empty export guard + rollback on swap failure
- Quiet mode (`-q`) for cron use
- Audit log to `/var/log/cleanspam.log`

Deployed `check-spamprobe` cron job (05:00 daily) to auto-detect and fix
corrupt databases before business hours.

## Decision: Rust rewrite

spamprobe is orphaned (2007), BDB is orphaned, both likely removed from
Debian 14 (forky). Assessed alternatives:

- **rspamd per-user Bayes:** Works but needs Redis (12+ GB for 600 users)
- **bogofilter-sqlite:** Also dying, single-word only (no bigrams)
- **Stalwart FTRL:** Built-in on mesh nodes, but not for Postfix/Dovecot
- **gonzofilter (Go):** Best accuracy in benchmarks, but Go GC overhead

Decided on a Rust implementation because:
1. Zero-overhead abstractions = smallest binary + lowest memory
2. SQLite bundled = single static binary, cross-compiles to OpenWrt
3. No GC = predictable latency for sieve pipe integration
4. mail-parser crate from Stalwart Labs = proven MIME parsing
5. Robinson-Fisher algorithm is ~150 lines, well-documented

## Key insight: memory per message

The critical architectural win over spamprobe:
- BDB: load entire database (up to 1.5 GB) per message
- SQLite: query only the ~1000 tokens in the message = ~100 KB I/O

This means a 1.5 GB well-trained database performs identically to a 5 MB
new database in terms of classification speed and memory usage.

## Research findings

- `mail-parser`: zero-dep, zero-copy, from Stalwart — perfect fit
- `rusqlite` + `bundled`: static SQLite, 40M+ downloads
- Robinson-Fisher chi-squared combining: ~150 lines of Rust
- Estimated binary: 3-5 MB stripped static musl
- Estimated RSS: 5-8 MB during classification
- aarch64-musl cross-compile: Tier 2 (straightforward)
- mips (older OpenWrt): Tier 3 (painful, probably unnecessary)
